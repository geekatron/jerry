## 2. The Framework (Verbose)

### Step 1 – Frame the Problem (5W1H)
Explain the problem so clearly a 5‑year‑old can repeat it.

Who, What, Where, When, Why, How
No solution work is allowed until this is complete.

### Step 2 – Classify the Problem
Misclassification guarantees failure.

- **Simple** → Known answer → best practices
- **Complicated** → Expert analysis
- **Complex** → Probe → sense → respond
- **Chaotic** → Act immediately → stabilize

### Step 3 – Diagnose
Choose the diagnostic tool:
- 5 Whys → linear
- Fishbone → multi-factor
- Abduction → unknown unknowns

### Step 4 – Ideate
Divergent thinking only.
No evaluation.
No feasibility discussion.

### Step 5 – Decide
Now converge.
Tradeoffs:
- Risk
- Cost
- Time
- Reversibility
- Impact

### Step 6 – Act
Execution with monitoring.
Be ready to abort.

### Step 7 – Verify & Learn
Confirm outcome.
Institutionalize learning.

## 4. Teaching Script (Live)

**Instructor:**
“Before solutions, tell me the story. Who is hurt? Where? When?”

**Pause.**

“If we misclassify this problem, what happens?”

**Pause.**

“Now — brainstorm only. No judging.”

**After ideas:**
“Which assumption would fail first?”

## 5. Failure Modes Appendix

### Common Failures
- Solving before framing
- Wrong problem classification
- No divergence
- No red team
- No learning loop

### Anti-Patterns
- Best practices in chaos
- Improvisation as permanent solution
- Consensus instead of correctness
- Blame instead of diagnosis
