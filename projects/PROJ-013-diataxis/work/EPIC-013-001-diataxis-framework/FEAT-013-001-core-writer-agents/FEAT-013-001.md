## Summary

Implement four specialized writer agents, one for each Diataxis quadrant: tutorial, how-to, reference, and explanation. Each agent follows the dual-file architecture (`.md` + `.governance.yaml`) per H-34 and includes quadrant-specific methodology, guardrails, and output templates.

**Value Proposition:**
- Each quadrant gets a purpose-built writer agent with quadrant-specific methodology
- Agents enforce Diataxis content boundaries (no mixing of quadrant concerns)
- Consistent output quality across all four documentation types

---

## Benefit Hypothesis

**We believe that** creating specialized writer agents for each Diataxis quadrant

**Will result in** documentation that correctly matches reader intent (learning vs. working vs. looking up vs. understanding)

**We will know we have succeeded when** each agent produces documentation that passes adversarial review and correctly adheres to its quadrant's constraints
