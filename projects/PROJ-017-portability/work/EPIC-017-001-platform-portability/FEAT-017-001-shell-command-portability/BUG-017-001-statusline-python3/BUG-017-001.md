## Summary

The `.claude/settings.local.json` statusLine hook hardcodes `python3` as the interpreter, which does not exist on standard Windows Python installations where the binary is named `python` or `python.exe`.

**Key Details:**
- **Symptom:** statusLine hook fails silently on Windows; no status displayed
- **Frequency:** Every invocation on Windows
- **Workaround:** Manually edit `settings.local.json` to use `python` instead of `python3`

---

## Steps to Reproduce

### Prerequisites

- Windows machine with Python installed via official installer (python.org)
- Jerry repository cloned

### Steps to Reproduce

1. Clone jerry repository on Windows
2. Run `uv sync` to install dependencies
3. Open Claude Code in the jerry directory
4. Observe that statusLine shows no output / errors

### Expected Result

statusLine displays current project and session status.

### Actual Result

statusLine fails because `python3` is not found in PATH on Windows.
