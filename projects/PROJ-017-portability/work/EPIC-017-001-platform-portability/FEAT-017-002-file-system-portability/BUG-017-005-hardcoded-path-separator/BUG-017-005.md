## Summary

Several source files construct paths using hardcoded `/` separators instead of `pathlib.Path` or `os.path.join()`. While Python generally handles forward slashes on Windows, edge cases exist with subprocess calls and external tools that expect native path separators.

**Key Details:**
- **Symptom:** Potential path resolution failures on Windows in subprocess or external tool invocations
- **Frequency:** Intermittent; depends on which code paths execute on Windows
- **Workaround:** None required for most Python operations; problematic only at subprocess boundaries

---

## Steps to Reproduce

### Prerequisites

- Windows machine
- Jerry repository cloned

### Steps to Reproduce

1. Search codebase for string concatenation path construction (e.g., `f"{dir}/{file}"`)
2. Identify instances passed to `subprocess.run()` or external tools
3. Test those code paths on Windows

### Expected Result

All paths constructed using `pathlib.Path` or `os.path.join()` for platform independence.

### Actual Result

Some paths use hardcoded `/` which may fail in subprocess calls on Windows.
