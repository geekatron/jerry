## Summary

Docstrings and inline comments reference `/tmp` as a temporary directory path. On Windows, the temp directory is accessed via `%TEMP%` environment variable (user-specific AppData\Local\Temp). Documentation should use `tempfile.gettempdir()` references for platform neutrality.

**Key Details:**
- **Symptom:** Misleading documentation for Windows developers; copy-pasted examples fail
- **Frequency:** Every documentation read on Windows
- **Workaround:** Developers mentally substitute with Windows temp path

---

## Steps to Reproduce

### Prerequisites

- Any platform; issue is in documentation content

### Steps to Reproduce

1. Search codebase for `/tmp` in docstrings and comments
2. Observe hardcoded Unix-specific path references

### Expected Result

Documentation references `tempfile.gettempdir()` or uses platform-neutral examples.

### Actual Result

Docstrings reference `/tmp` which is Unix-specific.
